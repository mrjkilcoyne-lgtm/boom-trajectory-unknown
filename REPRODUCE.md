# Reproducing this submission

Every output file in this repo (`prediction_submission.csv`, `design_submission.csv`, `self_score.json`) can be regenerated byte-for-byte from the challenge training data. RNG seed is hard-coded to 42.

## Prerequisites

- Go 1.22 or later (`go version` to check)
- The challenge data pack from Freelancer.com's Boom: Trajectory Unknown repo (provided to challenge participants): `train.csv`, `train_labels.csv`, `test.csv`, `constraints.json`

## Steps

```bash
# 1. Clone this repo
git clone https://github.com/mrjkilcoyne-lgtm/boom-trajectory-unknown.git
cd boom-trajectory-unknown

# 2. Drop the competition data into a `data/` directory at the repo root
mkdir -p data
cp /path/to/challenge/train.csv          data/
cp /path/to/challenge/train_labels.csv   data/
cp /path/to/challenge/test.csv           data/
cp /path/to/challenge/constraints.json   data/

# 3. The pipeline expects data/ to be a sibling of the src/ working directory.
#    Run it from the repo root:
cd src
go run .

# 4. Outputs land next to main.go in src/:
#    - prediction_submission.csv
#    - design_submission.csv
#    - self_score.json
#    - SUBMISSION_READY.md (summary the tool writes at the end)

# 5. Diff against the vendored copies at the repo root
diff prediction_submission.csv ../prediction_submission.csv
diff design_submission.csv     ../design_submission.csv
diff self_score.json           ../self_score.json
# All three should be empty.
```

## Expected runtime

On a modern laptop (8-core, SSD) the whole harness pipeline — load, 5-fold OOF across 4 adepts (RF+GBM+KNN+ridge), per-target simplex weight search, refit on full data, predict, inverse-design, write CSVs — completes in a few minutes. No GPU, no external dependencies, no network calls after `go mod download` (and the go.mod has zero non-stdlib imports anyway).

## What the pipeline prints

You'll see progress logs for each of the six targets showing the grid-search winner (ensemble weights + holdout NRMSE), then the composite score, then the inverse-design block reporting how many training rows satisfied the constraints and how many of those were selected for diversity.

## If something drifts

- Check Go version — map iteration order is stable under seeded RNG only when you control it; this pipeline does, but a Go point-release regression is the usual culprit if bytes differ.
- Check the input CSVs are unmodified from Freelancer's originals (byte-exact).
- If `self_score.json.timestamp` differs, that's expected — it's the only non-deterministic field. The NRMSE values and ensemble weights are deterministic.
